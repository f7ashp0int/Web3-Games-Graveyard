# Google Form Specification for Web3 Games Graveyard

This document outlines the structure and configuration for the "Web3 Games Graveyard - Game Submission" Google Form.

## Form Title and Description
- **Title**: Web3 Graveyard Lists
- **Public URL**: https://docs.google.com/forms/d/e/1FAIpQLScrRVspMxpfoFIBUoq474p-LzNSZyPt2CTqD34s9BmVqwdq5g/viewform?usp=dialog
- **Description**: Submit a dead or dying Web3 game to the graveyard. Please provide as much accurate information as possible.

## Form Fields

### Section 1: Basic Information

1. **Game Name** (Required)
   - Type: Short answer
   - Description: The official name of the game.

2. **Game Status** (Required)
   - Type: Dropdown
   - Options:
     - Dead (The game has officially shut down)
     - Dying (The game is active but development has ceased or player base has collapsed)
     - Ghost Town (The servers are up but there is no activity)

3. **Description** (Required)
   - Type: Paragraph
   - Description: A brief description of what the game was about.

4. **Reason for Demise** (Required)
   - Type: Paragraph
   - Description: Why did the game fail? (e.g., Rug pull, lack of funding, market conditions, etc.)

### Section 2: Timeline

5. **Launch Date**
   - Type: Date (or Short Answer if fuzzy dates are needed)
   - Description: When did the game launch? (Approximate is fine)

6. **Death Date**
   - Type: Date (or Short Answer if fuzzy dates are needed)
   - Description: When did the game shut down or cease operations?

### Section 3: Details & Metadata

7. **Logo URL**
   - Type: Short answer
   - Description: Direct link to the game's logo image (PNG/JPG).

8. **Category**
   - Type: Short answer
   - Description: e.g., MMORPG, FPS, Card Game, etc.

9. **Blockchain**
   - Type: Short answer
   - Description: Which blockchain(s) was the game on? (e.g., Ethereum, Solana, Polygon)

10. **Developer**
    - Type: Short answer
    - Description: Name of the development studio or team.

11. **Funding Raised**
    - Type: Short answer
    - Description: Total funding raised (e.g., "$5M", "Unknown").

12. **Peak Players**
    - Type: Short answer
    - Description: Estimated peak daily/monthly active users.

13. **Source / Reference**
    - Type: Short answer
    - Description: Link to an article, tweet, or announcement verifying the status.

14. **Tags**
    - Type: Short answer
    - Description: Comma-separated tags (e.g., P2E, NFT, Metaverse).

### Section 4: Contributor

15. **Your Name / Handle**
    - Type: Short answer
    - Description: Credit for the submission (optional).

## Settings
- **Collect email addresses**: Optional (useful for follow-up but might reduce submissions)
- **Allow response editing**: Yes
